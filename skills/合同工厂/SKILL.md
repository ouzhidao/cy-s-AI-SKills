## D.4 产品明细表结构

| 序号 | 品名 | 等级 | 规格 | 数量(吨) | 含税单价(元/吨) | 税率(%) | 含税总额 | 备注 |
|:---:|------|:---:|------|:---:|:---:|:---:|------|------|

汇总行：
- 总含税金额 / 货品不含税金额 / 增值税金额（9%）
- 备注行：生产日期/产地等

计算公式：
```
含税总额 = 数量 × 含税单价
不含税金额 = 含税总额 / 1.09
增值税 = 不含税金额 × 9%
```

## E.4 ⚠️ 重点提醒项（每次必须主动问）

### E.4.1 验收标准（最重要）
试用合同的验收标准必须明确，不同设备完全不同：
- **原则**：主动给出3~6条量化的、可操作的验收指标

- **如果用户拿不准**：从报价单/技术方案中提取关键参数，拟建议标准让用户确认

### E.4.2 问题处理机制（建议模板）

| 级别 | 处理 |
|:---:|------|
| 一般问题（非核心功能） | 供方免费维修，试用继续 |
| 关键问题（影响验收指标） | 供方整改，试用期重新计算 |
| 严重问题（2次不达标/15日无法整改） | 需方有权退货，供方承担往返费用 |
| 特殊工艺产品 | 双方现场沟通，需方配合至多一次调整测试 |

### E.4.3 质保期
- **不做默认假设**，每次主动问用户
- 可能是1年、2年、3年甚至5年，完全看设备类型

### E.4.4 收货人信息
- 提醒用户是否指定特定收货人+电话

## E.5 序号维护规则
在合同中插入或删除条款时：
- 检查插入位置的上下文序号体系
- 大标题用 `一、二、三`，子项用 `（1）（2）（3）`，再次级用 `① ② ③`
- 插入后确认前后序号连贯

## E.6 工作流程
1. 用户提需求 → 匹配卡片E
2. 确认甲方主体 + 乙方信息
3. 确认设备明细 + 价格
4. **⚠️ 主动建议验收标准**（按E.4.1）
5. **⚠️ 主动提出问题处理机制**（按E.4.2）
6. **⚠️ 主动问质保期**
7. **⚠️ 主动问收货人**
8. 确认交货天数 + 付款细节
9. 列出完整合同细节确认 → 生成
